## 🎨 出力フォーマット

### 1. 情報構造マップ（Information Architecture）
- サイト全体を3〜4階層で整理  
- 「学ぶ」「申し込む」「信頼を得る」「サポートを受ける」など、ユーザー意図ごとに分類

例：
```
├── ホーム（LP）
│   ├── 特徴・実績
│   ├── 学習カリキュラム
│   ├── 無料体験セミナー誘導
│   ├── 口コミ／メディア掲載
│   └── FAQ・CTA
├── カリキュラム詳細 (/school/kabu)
├── スケジュール／開催情報 (/school/schedule)
├── 教材紹介 (/open_campus/textbook/kabu)
├── 無料体験／セミナーLP (/seminar)
├── 受講生の声 (/voice)
├── 提携／メディア紹介 (/media)
├── ログイン／マイページ (/login)
└── 企業情報／お問い合わせ／法務 (/company, /contact, /privacy, /terms)
```

### 2. ページタイプ分類
| ページ名 | 目的 | 主導線（CTA） | 更新頻度 | CMS連携 |
|-----------|------|----------------|------------|-----------|
| ホーム | コンバージョン誘導 | 無料体験申込 | 中 | 一部 |
| カリキュラム詳細 | 学習内容理解 | 無料体験申込 | 低 | 可 |
| スケジュール | 日程確認 | 申込 | 高 | 必須 |
| 教材紹介 | 教材理解 | 資料請求 | 低 | 任意 |
| 無料体験LP | 登録誘導 | 申込完了 | 高 | 外部可 |
| 口コミ／メディア | 信頼醸成 | 二次CV | 中 | 可 |
| FAQ | 疑問解消 | 無料体験導線 | 中 | 可 |

  - 「無料体験申込」ボタン＋電話リンク（タップ計測）

---

### 4. 導線設計（User Flow）
1️⃣ トップ → カリキュラム概要を読む  
2️⃣ → 実績・口コミを確認  
3️⃣ → スケジュール確認 → 無料体験申込  
4️⃣ → 申込完了 → メール送信／サンクスページ  

🪜 **補助導線**  
- FAQ → 体験申込へ遷移  
- 口コミ／教材紹介 → 信頼強化 → LP再遷移  
- セミナー申込完了後 → LINEフォロー or メルマガ登録

---

### 5. コンテンツブロック定義
| コンテンツ | 構成要素 | 備考 |
|-------------|------------|------|
| ヒーロー | 見出し／サブ見出し／CTA | ファーストビューで申込誘導 |
| 実績 | 受講者数／満足度／提携実績 | 数値訴求 |
| カリキュラム | ステップ構造（15回） | タイムライン構成 |
| 口コミ | 星＋コメント＋写真 | スライダー形式 |
| スケジュール | 開催日・地域・残席 | CMS／API連携 |
| FAQ | 質問／回答／カテゴリ | Accordion UI |
| CTA | 常時配置（上中下） | CV率向上 |
