# 🎯 ステップ1：目的・要件定義（1日）

あなたはWebプロデューサー兼PdMです。
次の情報をもとに、**サイト制作の目的と要件を1日で整理するドキュメント**を作成してください。

---

## 🧠 入力情報
- サイトのテーマ：{例：株式投資スクール}
- 参考URL：{例：https://www.f-academy.jp/school/kabu.html}
- 想定ターゲット：{例：投資初心者〜中級者、20〜50代社会人}
- 主なコンバージョン：{例：無料体験セミナー申込}
- 運営形態：{例：自社主催オンラインスクール、オフライン併用}
- コンテンツ種別：{例：カリキュラム紹介／受講生の声／申込フォーム／FAQ}

---

## 🧩 出力フォーマット

### 1. 目的（Purpose）
- このサイトを通じて何を実現するか（数値KPIがあれば記載）
- 競合との差別化ポイント

### 2. ターゲット（Persona）
- 年齢層・属性・課題
- 来訪前の心理状態／求めている価値

### 3. コンバージョン目標（Goal）
- 一次CV（無料体験申込など）
- 二次CV（資料請求／SNSフォローなど）
- 遷移率・成約率のKPI（仮値）

### 4. 機能要件（Functional Requirements）
| 機能カテゴリ | 内容 | 備考 |
|---------------|------|------|
| LP構成 | ヒーロー、特徴、実績、CTA | トップページ |
| カリキュラム紹介 | 15回の学習ステップ | タイムライン形式 |
| スケジュール検索 | 日付・地域・オンライン | CMS連携想定 |
| 申込フォーム | 名前、メール、電話、希望日 | reCAPTCHA対応 |
| FAQ／口コミ | よくある質問・受講生の声 | CMS管理可能に |
| マイページ導線 | 既存システムへのリンク | 連携のみ可 |

### 5. 非機能要件（Non-functional）
- 速度：LCP < 2.5s
- SEO：title/desc最適化、構造化データ対応
- 保守：CMSで更新可能
- セキュリティ：reCAPTCHA、CSRF対策
- アクセシビリティ：WCAG 2.1 AA準拠を目標

### 6. 成功指標（KPI）
| 指標 | 目標値 | 測定方法 |
|------|--------|----------|
| 無料体験申込率 | 3〜5% | GA4 |
| LP滞在時間 | 60秒以上 | GA4 |
| モバイル速度スコア | 85以上 | PageSpeed |

### 7. リスク・留意点
- 金融関連のため表現規制あり（リスク表記・免責必須）
- スパム申込／誇大表現対策
- CMS更新ルール設計（権限管理）

### 8. 今後の確認アクション
- 想定ユーザーの検索キーワード調査
- 競合サイトベンチマーク（3社）
- 申込動線ABテスト案出し
