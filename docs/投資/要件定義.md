# 株式投資スクール Web サイト 要件定義書

**作成日：2025 年 10 月 31 日**  
**バージョン：1.0**  
**ステータス：初版ドラフト**

---

## 1. 目的（Purpose）

### サイトの実現目標

- **無料体験セミナー申込を月間 100 件獲得**（初年度目標）
- 投資初心者が安心して学べる環境を提供し、継続的な学習コミュニティを形成
- 既存受講生の満足度向上とクチコミによる自然流入の促進

### 競合との差別化ポイント

- **初心者に特化した 15 回の体系的カリキュラム**：他社は中級者向けが多い
- **実践的な投資シミュレーション環境**：座学だけでなく疑似体験が可能
- **オンライン・オフライン両対応**：場所を選ばず柔軟に受講できる
- **受講生コミュニティの活性度**：卒業後も情報交換できる仕組み

---

## 2. ターゲット（Persona）

### 主要ペルソナ

| 属性         | 詳細                                                      |
| ------------ | --------------------------------------------------------- |
| **年齢層**   | 20〜50 代（特に 30〜40 代のビジネスパーソン）             |
| **職業**     | 会社員（営業・事務・IT 系）、自営業、主婦層               |
| **年収**     | 400〜800 万円                                             |
| **投資経験** | 初心者〜中級者（NISA 口座は開設済みだが活用できていない） |

### 課題・心理状態

- **投資に興味はあるが何から始めればいいかわからない**
- NISA や iDeCo の制度は知っているが、銘柄選びに自信がない
- 独学では情報が多すぎて混乱している
- 高額な投資セミナーは怪しいと感じている（信頼性を重視）
- 仕事が忙しく、自分のペースで学びたい

### 求めている価値

- 体系的かつ実践的な学習プログラム
- 信頼できる講師陣とカリキュラム
- 無料体験で雰囲気を確認したい
- オンライン受講の柔軟性

---

## 3. コンバージョン目標（Goal）

### 一次 CV（Primary Conversion）

- **無料体験セミナー申込**
  - 目標 CVR：3〜5%（サイト訪問者ベース）
  - 月間目標：100 件
  - 申込完了後の実受講率：70%以上

### 二次 CV（Secondary Conversion）

- **資料請求・パンフレットダウンロード**（メールアドレス取得）
- **公式 LINE アカウント友だち登録**（定期的な情報発信）
- **SNS フォロー**（Instagram、X/Twitter）
- **ブログ・コラム記事の定期閲覧**（リピート訪問）

### KPI 設定（仮値）

| 指標                        | 目標値        | 備考           |
| --------------------------- | ------------- | -------------- |
| サイト訪問者数              | 月間 3,000 人 | SEO・広告流入  |
| LP→ 申込フォーム遷移率      | 15%           | CTA の効果測定 |
| 申込フォーム完了率          | 60%           | EFO 対策必須   |
| 体験セミナー → 本申込転換率 | 30%           | オフライン施策 |

## 4. 機能要件（Functional Requirements）

| 機能カテゴリ                | 内容                                                                                                   | 備考                              |
| --------------------------- | ------------------------------------------------------------------------------------------------------ | --------------------------------- |
| **LP 構成（トップページ）** | ヒーロービジュアル、スクールの特徴（3 つの強み）、カリキュラム概要、受講生の声、講師紹介、無料体験 CTA | スクロールに応じて CTA 表示       |
| **カリキュラム詳細ページ**  | 15 回の学習ステップをタイムライン形式で表示、各回の学習内容・到達目標・使用ツール                      | アコーディオンまたはタブ UI       |
| **スケジュール・開催情報**  | 日付・地域（東京/大阪/オンライン）・時間帯で絞り込み検索                                               | CMS 連携（Strapi/WordPress 想定） |
| **無料体験申込フォーム**    | 氏名、メールアドレス、電話番号、希望日時、参加形態（オンライン/オフライン）、質問・要望（任意）        | reCAPTCHA v3、確認画面あり        |
| **受講生の声（実績紹介）**  | 顔写真・年齢・職業・受講前後の変化をカード形式で表示                                                   | 5〜10 件掲載、CMS 管理            |
| **FAQ（よくある質問）**     | カテゴリ別（料金・受講形式・カリキュラム・サポート）にアコーディオン表示                               | 検索機能（将来的に）              |
| **ブログ・コラム**          | 投資の基礎知識、市場ニュース解説、受講生インタビュー                                                   | SEO 対策コンテンツ                |
| **マイページ導線**          | 既存受講生向けログインページへのリンク（外部システム連携）                                             | 認証は既存システム利用            |
| **お問い合わせフォーム**    | 一般的な問い合わせ用（申込とは別）                                                                     | Google フォーム埋め込みも可       |

## 5. 非機能要件（Non-functional Requirements）

### パフォーマンス

- **LCP（Largest Contentful Paint）**：2.5 秒以内
- **FID（First Input Delay）**：100ms 以内
- **CLS（Cumulative Layout Shift）**：0.1 以下
- モバイルページ速度スコア：85 以上（PageSpeed Insights）

### SEO 対策

- title/meta description 最適化（各ページ固有）
- 構造化データ対応（Course、Organization、FAQPage）
- XML サイトマップ自動生成
- OGP 設定（SNS シェア対応）
- 内部リンク最適化

### 保守性

- **CMS 導入**：非エンジニアでも更新可能（Strapi/WordPress/Contentful）
- スケジュール・受講生の声・FAQ は CMS 管理
- ブログ記事の投稿・編集機能
- 画像アップロード・リサイズ自動化

### セキュリティ

- **フォーム保護**：reCAPTCHA v3 実装
- CSRF 対策（トークン検証）
- SSL/TLS 暗号化（必須）
- 個人情報の暗号化保存
- セキュリティヘッダー設定（CSP、X-Frame-Options 等）

### アクセシビリティ

- **WCAG 2.1 AA 準拠**を目標
- キーボード操作対応
- スクリーンリーダー対応（aria-label、alt 属性）
- 色のコントラスト比 4.5:1 以上
- フォントサイズ調整可能

### 対応環境

- **ブラウザ**：Chrome/Safari/Edge/Firefox 最新 2 バージョン
- **デバイス**：PC、タブレット、スマートフォン（レスポンシブデザイン）
- **OS**：Windows、macOS、iOS、Android

## 7. リスク・留意点

### 法規制・コンプライアンス

- **金融商品取引法**：投資助言業の登録有無を確認
- **景品表示法**：「必ず儲かる」等の誇大表現は厳禁
- **リスク表記**：投資にはリスクが伴うことを明記（免責事項ページ必須）
- **個人情報保護法**：プライバシーポリシー整備、同意取得

### スパム・セキュリティ対策

- 申込フォームへのスパム対策（reCAPTCHA、ハニーポット）
- SQL インジェクション・XSS 対策
- DDoS 攻撃対策（CDN 利用検討）

### CMS 運用リスク

- 更新権限管理の設計（管理者・編集者・閲覧者の役割分担）
- 誤操作による公開情報の削除・改変防止（バージョン管理）
- 画像サイズ・フォーマットのガイドライン策定

### ブランド毀損リスク

- 受講生の声の真実性担保（虚偽は厳禁）
- 講師プロフィールの定期更新
- 古い情報の放置による信頼性低下防止

## 8. 今後の確認アクション

### フェーズ 1：情報収集（1 週間）

- [ ] **想定ユーザーの検索キーワード調査**
  - Google キーワードプランナー、ラッコキーワード活用
  - 「株式投資 初心者 スクール」「NISA 勉強」等のボリューム確認
- [ ] **競合サイトベンチマーク（3 社以上）**
  - ファイナンシャルアカデミー、マネーセミナー、他オンラインスクール
  - 強み・弱み、UI/UX、価格設定を分析
- [ ] **既存受講生インタビュー（3〜5 名）**
  - 申込の決め手、不安だった点、改善要望をヒアリング

### フェーズ 2：設計準備（2 週間）

- [ ] **申込動線 AB テスト案出し**
  - CTA 位置（ヘッダー固定 vs フローティング）
  - フォーム項目数（最小限 vs 詳細情報取得）
  - 無料体験の訴求文言バリエーション
- [ ] **コンテンツ素材収集**
  - 受講生の声（写真・インタビュー）
  - 講師プロフィール・写真
  - カリキュラム詳細資料
- [ ] **CMS 選定・見積もり取得**
  - WordPress（プラグイン構成検討）
  - Strapi（HeadlessCMS）
  - Contentful（有料だが柔軟性高い）

### フェーズ 3：開発前最終確認（1 週間）

- [ ] **ワイヤーフレーム作成・レビュー**
- [ ] **デザインコンセプト決定**（配色・トーン）
- [ ] **外部連携システムの仕様確認**（既存マイページ、決済システム）
- [ ] **GA4 タグ・GTM 設定計画**
